# Product Requirement Document (PRD)

**Project:** Furina Advocate Simulator (가제)

**Version:** 1.0 (MVP)

**Author:** 임세훈

**Target Environment:** Modern Terminal Emulators (Kitty, Ghosty, Windows Terminal via PowerShell) with Kitty Graphics Protocol / Sixel support.

---

## 1. 개요 및 목표 (Introduction & Objectives)

### 1.1 제품 개요

본 제품은 현대적 터미널 에뮬레이터 환경에서 구동되는 하이엔드 텍스트/그래픽 융합형 유저 인터페이스(TUI) 기반의 서브컬처 게임이다. 우마무스메 구조의 파라미터 육성 시뮬레이션, 젠존제 스타일의 사선 분할 레이아웃 재판 배틀, 그리고 대형 언어 모델(LLM) 및 고해상도 그래픽 스트리밍이 결합된 미연시 세션을 제공한다.

### 1.2 개발 목적

* 레거시 TUI의 텍스트 배치 한계를 극복하고, 호요버스 스타일의 정돈된 플랫 디자인을 터미널 에뮬레이터 환경에 자원 효율적으로 구현한다.
* 기존 개발 자산(termplay, Gascii)의 비디오/이미지 버퍼 제어 로직을 통합하여 최적의 성능(60 FPS)을 보장하는 차세대 TUI 게임 아키텍처를 검증한다.

## 2. 타겟 환경 및 시스템 사양 (Target Environment Specs)

### 2.1 호환 터미널 에뮬레이터 명세

본 소프트웨어는 로우 레벨 이미지/비디오 버퍼 인젝션을 위해 특정 그래픽 프로토콜이 지원되는 환경으로 실행을 제한한다. 레거시 CMD 및 기본 xterm 환경은 지원 대상에서 명시적으로 제외한다.

| 터미널 에뮬레이터 | 지원 프로토콜 | 비고 |
| --- | --- | --- |
| **Kitty** | Kitty Graphics Protocol | 기본 타겟 및 개발 환경 |
| **Ghosty** | Kitty Graphics Protocol / Sixel | 차세대 렌더러 지원 환경 |
| **Windows Terminal** | Sixel | PowerShell 구동 환경 필수 |

### 2.2 기존 모듈 재사용 계획 (System Integration)

* **termplay 모듈 이식:** 미연시 페이즈의 3D 리액션 비디오 버퍼 스트리밍 엔진으로 활용.
* **Gascii 모듈 이식:** 재판 페이즈의 2D 캐릭터 컷인 및 배경 픽셀 변환 매핑 엔진으로 활용.

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 FR-1: 시스템 초기화 및 진입 (Splash Screen)

* **상세:** 게임 실행 시 그래픽 프로토콜 활성화 여부를 감지하고 엔진 버퍼를 초기화한다.
* **UI/UX 사양:** 붕괴: 스타레일 스타일의 미니멀한 기하학적 패턴 배치 및 인트로 애니메이션 표출.

### 3.2 FR-2: 육성 시뮬레이션 엔진 (Training Phase)

* **상세:** 주차별(Turn) 제한된 행동력을 소모하여 피고인의 5대 파라미터를 육성하는 루프를 제공한다.
* **데이터 모델 (5대 파라미터):**
* `Logic Speed` (논리 속도): 반론 전개 속도 및 모순 포착 확률 보정.
* `Mental Stamina` (멘탈 유지력): 재판 중 데미지 제어 및 HP 유지.
* `Speech Power` (압박 화술): 변론 시 판세 주도권 및 타격 치명타 확률.
* `Guts` (배짱 지수): 유죄 위기 도래 시 고유 역전 스킬 발동 확률.
* `Intellect` (법률 지식): 패시브 스킬 보너스 및 영수증/증거물 매칭 정확도.


* **UI/UX 사양:** 붕괴: 스타레일 캐릭터 관리 인터페이스 벤치마킹. 좌측 세로형 내비게이션 탭, 중앙 가로형 수평 게이지 스탯 바, 우측 플랫 카드 형태의 스케줄 선택기 제공.

### 3.3 FR-3: 사선 분할 재판 배틀 (Court Phase)

* **상세:** 육성된 파라미터 수치를 기반으로 상대 검사와 변론 공방을 벌이는 자동 레이스 시스템을 구동한다.
* **전투 로직:** 텍스트 기반 변론 로그가 누적되며, 주사위 연산(스탯 반영) 결과에 따라 실시간으로 아군/적군의 판세 게이지가 변동한다. 임계점 도달 시 고유 스킬이 발동된다.
* **UI/UX 사양:** 젠존제(ZZZ) 코믹스 컷인 스타일 구현. `Gascii` 코어를 이용해 사선(`transform: skewX`) 형태로 분할된 프레임 영역을 확보하고, 검사와 푸리나의 고해상도 이미지를 비대칭 레이아웃으로 출력한다. 타격 타이밍에 "이의있소!" 대형 오버레이 팝업을 연출한다.

### 3.4 FR-4: LLM 연동 대화 콘솔 (Dating Phase)

* **상세:** 재판 결과 콘텍스트를 주입받은 LLM이 푸리나의 페르소나를 동적으로 연기하며 유저와 자유 대화를 진행한다.
* **인터랙션 로직:** 고정된 선택지 없이 유저가 인풋 폼에 직접 문자열을 입력하면, LLM API를 호출하여 청크(Chunk) 단위로 대사 버퍼를 받아와 실시간 스트리밍 출력한다.
* **UI/UX 사양:** 원신풍 반투명 플로팅 대화 상자 구조.
* 상단 70%: `termplay` 로직 기반으로 렌더링된 푸리나 리액션 비디오/와이어프레임 뷰포트 배치.
* 하단 30%: 다이아몬드 지시자가 포함된 네임태그 대화 상자 및 프롬프트 커맨드 입력 필드 통합.

## 4. 비기능 및 기술 요구사항 (Non-Functional Requirements)

### 4.1 성능 및 자원 관리 (Performance & Resource)

* **프레임 레이트:** 비디오 스트리밍 및 공방 애니메이션 구동 시 실시간 60 FPS 유지를 목표로 설정한다.
* **메모리 최적화:** 가상 돔(Virtual DOM) 방식의 불필요한 전체 리렌더링을 지양하며, 문자 단위 렌더링을 차단하고 텍스트 스로틀링(Throttle) 버퍼 제어를 통해 VRAM 및 RAM 누수를 제어한다. (RAM 50MB 이하 유지 가이드)
* **비동기 런타임:** 키보드 인터럽트 처리, 비디오 버퍼 스트리밍, LLM API HTTP 요청 처리가 상호 블로킹(Blocking)을 일으키지 않도록 멀티스레드(`tokio` 런타임 환경 등) 비동기 처리를 아키텍처 표준으로 삼는다.

### 4.2 화면 그래픽스 제어 사양 (Graphics Layering)

* **멀티 레이어 컴포지션:** 화면 출력 시 배경 이미지 레이어, 캐릭터 독립 레이어, UI 패널 레이어, 최상단 텍스트 레이어를 분리하여 메모리 상에서 병합 후 출력하는 구조를 채택한다.
* **인터페이스 스타일:** 외곽선 기호 기반의 해커/사이버펑크 연출을 전면 배제하고, 배경색 투명도 제어 및 유니코드 실선을 활용해 일반 웹 프론트엔드 레벨의 깔끔한 플랫 디자인 품질을 유지한다.

## 5. 개발 로드맵 및 MVP 마일스톤 (Milestones)

### Phase 1: 기술 검증 (PoC)

* **태스크 1:** 지정 터미널(Kitty/Ghosty) 내에서 `Gascii` 로직을 응용한 사선 분할 그래픽 버퍼 출력 및 프레임 드랍 여부 검증.
* **태스크 2:** `termplay` 비디오 버퍼 스트리밍 레이어 위에 `ratatui` 계열 텍스트 레이어를 깜빡임 없이 안정적으로 오버레이 합성하는 기술 검증.

### Phase 2: 최소 기능 구현 (MVP Release)

* **태스크 1:** 데이터 모델(`GameState`) 구조 확립 및 [1턴 훈련 ➔ 1판 자동 재판 ➔ 1회 LLM 세션 데이터 인젝션] 코어 루프 연동.
* **태스크 2:** LLM 응답 텍스트 스트리밍 수신 시 UI 레이아웃의 비정상적 깨짐 방지를 위한 뷰포트 고정 및 스크롤 버퍼 모듈 예외 처리 완료 후 배포.
