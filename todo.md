터미널 RGB 영상은 CPU 많이 먹는 게 거의 구조적 한계입니다. 매 프레임마다:

  1. ffmpeg가 프레임 디코딩
  2. Rust가 터미널 크기로 리사이즈
  3. SVG 오버레이 합성
  4. 셀마다 truecolor escape sequence 생성
  5. 터미널 앱이 그걸 다시 렌더링

  이 전체가 CPU를 씁니다. 특히 4, 5번이 큽니다.

  현실적인 선택지는 이 정도입니다.

  - ASCII/ANSI 모드로 전환: 제일 효과 큼. 색을 포기하거나 제한하면 출력량이 확 줄어
    듭니다.

  - RGB는 splash 짧게만 사용: 지금처럼 타이틀용이면 몇 초만 돌리고 다음 UI로 넘기는
    식.

  - 저해상도 RGB: 중앙 배율 버그 없이 하려면 “전체 셀 상한” 말고 “명시적 scale
    factor”를 둬야 합니다.

  - 프레임 캐싱: 영상이 짧고 고정이면 시작 전에 터미널 프레임 문자열로 미리 변환.
    재생 중 CPU는 줄지만 메모리와 로딩 시간이 늘어납니다.

  - Kitty/iTerm 이미지 프로토콜 사용: 터미널이 지원하면 가장 좋아요. RGB 텍스트 렌
    더 대신 이미지 프레임을 보내는 방식인데 호환성이 떨어집니다.

  내 추천은 다음 단계에서 모드 분리입니다.

  --rgb-splash-demo      고품질, CPU 높음
  --ascii-splash-demo    저부하, 전역 아스키 렌더 기반

  그리고 기본 splash는 ASCII/ANSI 쪽으로 두는 게 맞습니다. 게임이 터미널 기반이면
  이게 정체성에도 더 맞고, CPU도 훨씬 덜 먹습니다.
